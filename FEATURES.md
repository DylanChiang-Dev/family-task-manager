## 🏢 多團隊架構

### Slack/Feishu 風格的工作區管理

**核心理念**：每個用戶可以加入多個團隊（工作區），類似於 Slack 或飛書的架構，適合家庭和多個工作場景。

#### ✨ 主要特性

**🏗️ 多團隊支持**
- 一個賬號可管理多個團隊（家庭、工作、項目組等）
- 每個團隊的數據完全隔離，確保隱私安全
- 頂部下拉菜單一鍵切換團隊上下文
- 自動記住用戶的最後活動團隊

**🔗 邀請碼系統**
- 6位數字邀請碼（排除易混淆字符：0, O, I, 1）
- 管理員可隨時重新生成邀請碼
- 邀請碼唯一性校驗，防止衝突
- 支持通過邀請碼快速加入團隊

**👥 角色管理**
- **管理員（Admin）**：創建/管理團隊、管理成員、管理類別
- **成員（Member）**：創建任務、查看任務、更新任務狀態
- 權限分離，確保團隊管理安全有序

**🔄 團隊切換**
- 無縫切換團隊，界面數據即時更新
- 保持當前篩選狀態和視圖偏好
- 跨團隊操作時自動驗證權限

#### 📱 使用場景

**家庭場景**
```
家庭團隊：張家
├── 爸爸（管理員）
├── 媽媽（管理員）
├── 小明（成員）
└── 小紅（成員）

任務分類：家務、購物、學習、健康
```

**工作場景**
```
工作團隊：產品組
├── 產品經理（管理員）
├── 設計師（成員）
├── 前端開發（成員）
└── 後端開發（成員）

任務分類：開發、設計、測試、文檔
```

## 🔄 週期任務

### 智能化的重複任務管理

支援三種任務類型，滿足不同週期性需求。

#### ✨ 任務類型

**📌 一般任務**
- 一次性任務，完成後消失
- 適合臨時、非重複性工作
- 傳統任務管理模式

**🔄 週期任務**
- 按固定週期重複執行
- 自動生成下一週期實例
- 適用於常規性工作

**🔁 重複任務**
- 按條件重複的任務
- 完成後可手動創建下一個
- 適用於不定期重複工作

#### 📅 週期配置

**每日任務**
```
場景：每日打卡、日報
配置：每天執行一次
示例：
- 每日健身記錄
- 每日工作日報
- 每日飲水提醒
```

**每週任務**
```
場景：週會、週末大掃除
配置：選擇星期幾（多選）
示例：
- 每週一、三、五健身
- 每週六家庭聚會
- 每週日週報總結
```

**每月任務**
```
場景：月度總結、賬單支付
配置：選擇每月日期（多選，逗號分隔）
示例：
- 每月1號支付房租
- 每月15號發工資
- 每月25號月度總結
```

**每年任務**
```
場景：年度回顧、生日紀念
配置：選擇月份和日期
示例：
- 每年1月1日新年計劃
- 每年生日紀念日
- 每年12月31日年度回顧
```

#### 🎯 智能生成

**自動創建下一週期**
- 任務完成後自動檢查是否為週期任務
- 根據週期配置計算下一個執行時間
- 自動創建新任務實例
- 保留原任務的所有屬性

**配置靈活性**
- 支持週期任務的截止日期配置
- 支持週期任務的分配人設置
- 支持週期任務的優先級配置
- 支持編輯已創建的週期任務

## 🌙 農曆日期

### 本地化農曆轉換系統

純 JavaScript 實現，無需網絡請求，支持 1900-2100 年共 200 年的農曆日期轉換。

#### ✨ 核心功能

**🗓️ 精確轉換**
- 公曆日期 → 農曆日期
- 農曆日期 → 公曆日期
- 支持閏月計算
- 支持二十四節氣

**📅 日曆顯示**
- 主日曆顯示公曆日期
- 副標顯示對應農曆日期
- 農曆節日自動標記
- 閏月特殊顯示

**🎉 節日提醒**
- 傳統節日：春節、清明、端午、中秋等
- 現代節日：元旦、勞動節、國慶等
- 自定義節日支援
- 節日前置提醒

#### 📋 支持範圍

**時間範圍**
- 起始：1900年1月31日（農曆1900年正月初一）
- 結束：2100年12月31日
- 精度：完整200年數據

**地區適配**
- 中國大陸節日
- 港澳台節日
- 海外華人節日
- 可自定義節日

#### 🔧 技術實現

**算法特點**
- 基於天文算法的高精度轉換
- 使用農曆信息數組壓縮存儲
- 純前端實現，無後端依賴
- 快速查詢，即時響應

**性能優化**
- 數據壓縮：僅 8KB 數組文件
- 計算優化：毫秒級轉換速度
- 快取機制：減少重複計算
- 按需加載：節省內存占用
